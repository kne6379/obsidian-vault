# Claude 작업 가이드 (공통)

이 문서는 AI(Claude)가 Obsidian Vault 전체에서 작업할 때 **가장 먼저 읽어야 할 공통 지침서**입니다.

---

## 볼트 목적

이 볼트는 **개인 지식 연결 그래프(Knowledge Graph)를 구성하기 위한 Obsidian 문서 저장소**입니다. 단순히 정보를 나열하는 것이 아니라, 개념 간의 관계를 `[[링크]]`로 연결하여 지식의 네트워크를 형성하는 것이 목표입니다.

---

## 볼트 구조

```
Obsidian Vault/
├── CLAUDE.md            ← 현재 문서 (공통 지침)
├── develop/             # 개발 지식 저장소
│   ├── _meta/           # 메타 문서 (RULES, INDEX, GLOSSARY)
│   ├── _templates/      # 문서 템플릿
│   ├── concepts/        # 개념/이론 문서
│   ├── tutorials/       # 실습/가이드 문서
│   ├── troubleshooting/ # 문제 해결 기록
│   └── references/      # 외부 참고 자료
└── work/                # 회사 프로젝트 저장소
    ├── _meta/           # 메타 문서 (RULES, INDEX, GLOSSARY)
    ├── _templates/      # 문서 템플릿
    ├── projects/        # 프로젝트별 문서
    ├── decisions/       # 기술 의사결정 기록
    ├── troubleshooting/ # 문제 해결 기록
    └── references/      # 외부 참고 자료
```

각 영역의 세부 규칙은 해당 폴더의 `_meta/RULES.md`를 참조합니다.

## 공통 작성 규칙

### 문체

- **"~니다" 체의 공식 문어체**를 사용합니다.
- 구어체, 반말, 이모지는 사용하지 않습니다.

### 용어

- **한글 우선 원칙**: 의미 전달에 불필요한 영어 단어의 괄호 병기는 지양합니다.
- **영문 병기 기준**: 최초 등장 시 개념 정의가 필요한 경우에만 `한글(English)` 형식으로 병기합니다.
- 이후 동일 문서 내에서는 한글만 사용합니다.

### 분량 및 명확성

- 핵심 정보가 누락되지 않도록 하되, 불필요하게 장황한 설명은 피합니다.
- 추측이나 모호한 표현은 피하고, 정의/구분/비교가 필요한 경우 명확히 구분합니다.

### 품질 기준

- 사내 보고서 또는 실무 공유 문서에 바로 사용할 수 있는 수준으로 작성합니다.

## 공통 프론트매터 형식

모든 문서 최상단에 포함:

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [태그1, 태그2]
status: (영역별 상태값 참조)
---
```

---

## 공통 링크 규칙

### 내부 링크

- 다른 문서에서 정의된 개념은 **`[[문서명]]`** 형식으로 링크합니다.
- 특정 섹션 링크: `[[문서명#섹션명]]`
- 별칭 사용: `[[문서명|표시할 텍스트]]`

### 관련 문서 섹션

모든 문서 하단에 `## 관련 문서` 섹션을 추가하고, 관련 문서들을 링크합니다.

### 외부 링크

외부 참고 자료는 `## 참고 자료` 섹션에 별도 정리합니다.

---

## 작업 전 필수 확인

1. **이 문서** - 공통 규칙 확인
2. **해당 영역의 `_meta/RULES.md`** - 세부 규칙 확인
3. **해당 영역의 `_meta/INDEX.md`** - 기존 문서 확인 (중복 방지)
4. **해당 영역의 `_meta/GLOSSARY.md`** - 용어 확인
5. **해당 영역의 `_templates/`** - 템플릿 활용

---

## 향후 확장 계획

현재는 개발 지식과 회사 프로젝트에 집중하고 있으나, 이후 삶 전반의 지식으로 확장할 계획입니다.

- **생활/정책**: 국가 정책, 청약 정보, 세금/재테크
- **취미/실용**: 요리, 건강, 생산성
- **인문/사고**: 철학, 가치관, 의사결정 프레임워크

확장 시 동일한 구조(`_meta`, `_templates` 등)를 각 영역별로 생성하여 일관성을 유지합니다.
