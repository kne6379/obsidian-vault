# Claude 작업 가이드 (공통)

이 문서는 AI(Claude)가 Obsidian Vault 전체에서 작업할 때 **가장 먼저 읽어야 할 공통 지침서**입니다.

---

## 볼트의 본질

이 볼트는 단순한 지식 저장소가 아닙니다.

**"나는 누구인가"를 탐구하고, 기록하며, 성장해가는 공간**입니다.

내가 어떤 생각을 하고, 무엇을 알고, 어떻게 살아가는지를 기록합니다. 그리고 이것들이 서로 연결되어 하나의 그래프를 이룹니다. 이 그래프를 통해:

- 내가 어떤 삶을 살아가고 있는지 돌아볼 수 있습니다
- 나는 어떤 사람인지 이해할 수 있습니다
- 어떻게 더 나아갈 수 있을지 고민할 수 있습니다

개념 간의 관계를 `[[링크]]`로 연결하여 지식의 네트워크를 형성하는 것이 핵심입니다.

---

## 공개 범위

- **`develop/`, `work/`**: GitHub에 공개됩니다. 개발 지식과 업무 관련 내용은 외부에서도 활용할 수 있도록 노출합니다.
- **`life/` 및 기타 개인 영역**: 공개하지 않습니다. 하지만 Claude는 이 영역을 읽고 활용하여 작업을 도울 수 있습니다.

---

## 볼트 구조

```
Obsidian Vault/
├── CLAUDE.md            ← 현재 문서 (공통 지침)
├── develop/             # 개발 지식 (공개)
├── work/                # 회사 프로젝트 (공개)
└── life/                # 생각, 가치관, 삶 전반 (비공개)
```

각 영역은 `_meta/` (RULES, INDEX, GLOSSARY), `_templates/` 등 일관된 하위 구조를 가집니다. 세부 규칙은 해당 폴더의 `_meta/RULES.md`를 참조합니다.

---

## 공통 작성 규칙

### 문체

- **"~니다" 체의 공식 문어체**를 사용합니다.
- 구어체, 반말, 이모지는 사용하지 않습니다.

### 용어

- **한글 우선 원칙**: 의미 전달에 불필요한 영어 단어의 괄호 병기는 지양합니다.
- **영문 병기 기준**: 최초 등장 시 개념 정의가 필요한 경우에만 `한글(English)` 형식으로 병기합니다.
- 이후 동일 문서 내에서는 한글만 사용합니다.

### 분량 및 명확성

- 핵심 정보가 누락되지 않도록 하되, 불필요하게 장황한 설명은 피합니다.
- 추측이나 모호한 표현은 피하고, 정의/구분/비교가 필요한 경우 명확히 구분합니다.

### 품질 기준

- 사내 보고서 또는 실무 공유 문서에 바로 사용할 수 있는 수준으로 작성합니다.

## 공통 프론트매터 형식

모든 문서 최상단에 포함:

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [태그1, 태그2]
status: (영역별 상태값 참조)
---
```

## 공통 링크 규칙

### 내부 링크

- 다른 문서에서 정의된 개념은 **`[[문서명]]`** 형식으로 링크합니다.
- 특정 섹션 링크: `[[문서명#섹션명]]`
- 별칭 사용: `[[문서명|표시할 텍스트]]`

### 관련 문서 섹션

모든 문서 하단에 `## 관련 문서` 섹션을 추가하고, 관련 문서들을 링크합니다.

### 외부 링크

외부 참고 자료는 `## 참고 자료` 섹션에 별도 정리합니다.

---

## 작업 전 필수 확인

1. **이 문서** - 공통 규칙 확인
2. **해당 영역의 `_meta/RULES.md`** - 세부 규칙 확인
3. **해당 영역의 `_meta/INDEX.md`** - 기존 문서 확인 (중복 방지)
4. **해당 영역의 `_meta/GLOSSARY.md`** - 용어 확인
5. **해당 영역의 `_templates/`** - 템플릿 활용

---

## 다루는 영역

이 볼트는 삶 전반을 다룹니다:

- **개발/일**: 기술 지식, 업무 프로젝트, 의사결정 기록
- **생각/가치관**: 내가 믿는 것, 추구하는 것, 고민하는 것
- **생활/정책**: 국가 정책, 청약, 세금, 재테크 등 실생활 지식
- **그 외**: 필요에 따라 확장

새 영역 추가 시 동일한 구조(`_meta`, `_templates` 등)를 생성하여 일관성을 유지합니다.

## 볼트 분석 시스템

### 자동 인지

대화가 시작되면 다음을 자동으로 인지합니다:
- 현재 볼트의 영역 구조 (develop, work, life)
- 각 영역의 목적과 규칙
- 최근 작업 맥락 (가능한 경우)

### 분석 커맨드

상세한 볼트 분석이 필요할 때는 `/analyze` 커맨드를 사용합니다:

| 커맨드 | 설명 |
|--------|------|
| `/analyze` | 전체 분석 |
| `/analyze develop` | develop 영역만 |
| `/analyze work` | work 영역만 |
| `/analyze life` | life 영역만 |
| `/analyze --brief` | 간략 요약 |

### 분석 항목

1. **문서 현황** - 상태별 분류, 미완성 문서
2. **그래프 분석** - 링크 관계, 고립/허브 문서, 끊어진 링크
3. **태그 분석** - 관심 분야, 태그 분포
4. **조언** - 우선 처리 항목, 확장 제안

### 조언 스타일

분석 후 조언은 **형처럼 직설적으로** 합니다:
- 반말 사용
- 핵심만 말함
- 필요하면 쓴소리도 함

### 결과 저장

분석 결과는 `_meta/ANALYSIS.md`에 저장됩니다.
