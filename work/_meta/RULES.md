# 회사 프로젝트 문서 작성 규칙

이 문서는 `work` 폴더 내 문서를 작성하거나 수정할 때 따라야 할 세부 규칙을 정의합니다.
공통 규칙(문체, 용어, 프론트매터, 링크 등)은 루트 `CLAUDE.md`를 따릅니다.

---

## 1. 문서 유형별 구조

### 1.1 프로젝트 문서 (projects/)

프로젝트의 전체 맥락을 기록하는 문서입니다.

1. **개요** - 프로젝트 목적, 배경, 기간
2. **기술 스택** - 사용 기술과 선택 사유
3. **아키텍처** - 시스템 구조, 주요 컴포넌트
4. **주요 기능** - 핵심 기능 설명
5. **이슈 및 회고** (선택) - 프로젝트 중 발생한 주요 이슈, 배운 점
6. **관련 문서**

### 1.2 기술 의사결정 기록 (decisions/)

기술 선택의 배경과 근거를 보존하는 문서입니다.

1. **배경** - 의사결정이 필요했던 상황
2. **검토 대안** - 비교한 선택지와 각각의 장단점
3. **최종 결정** - 선택한 대안과 그 사유
4. **관련 문서**

### 1.3 노트 (notes/)

미팅 노트, 리서치 정리 등 공유 목적의 문서입니다. 태그로 미팅/리서치를 구분합니다.

1. **요약** - 핵심 내용 요약
2. **상세 내용** - 논의 사항, 조사 결과 등
3. **액션 아이템** (선택) - 후속 작업이 있는 경우
4. **관련 문서**

### 1.4 문제 해결 기록 (troubleshooting/)

1. **문제 상황** - 에러 메시지, 증상
2. **원인 분석** - 왜 발생했는가
3. **해결 방법** - 구체적 해결 단계
4. **예방 방법** (선택) - 재발 방지책

---

## 2. 상태값

| 상태 | 설명 |
|------|------|
| `draft` | 초안 작성 중 |
| `active` | 진행 중인 프로젝트 |
| `completed` | 완료된 프로젝트 |
| `archived` | 더 이상 관리하지 않는 문서 |

---

## 3. 태그 체계

**유형**: `project`, `decision`, `note`, `troubleshooting`, `reference`

**노트 세부**: `meeting`, `research`

**상태**: `draft`, `active`, `completed`, `archived`

---

## 4. 문서 작성 워크플로우

1. 해당 유형의 섹션 구조에 따라 내용 작성
2. 관련 문서를 `[[링크]]`로 연결
3. [[INDEX]]에 새 문서 등록
