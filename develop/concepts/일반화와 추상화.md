# 일반화와 추상화

> 객체지향 설계의 핵심 개념으로, 일반화는 공통점을 추출하여 상위 개념을 형성하고, 추상화는 복잡성을 숨기고 핵심만 노출하는 과정입니다.

---

## 1. 정의

### 일반화 (Generalization)

여러 구체적인 대상들로부터 **공통점을 추출**하여 상위 개념을 형성하는 과정입니다. 개별 요소에서 일반적인 것으로 올라가는 **상향식 사고**입니다.

**핵심 질문:** "이것들은 같은 종류인가?"

### 추상화 (Abstraction)

복잡한 내부 구현은 숨기고 사용자에게 필요한 **핵심만 남겨 단순화**하는 과정입니다. 사용자 관점에서 필요한 정보만 노출하는 **하향식 사고**입니다.

**핵심 질문:** "사용자에게 필요한 것이 무엇인가?"

---

## 2. 비교

| 구분 | 일반화 | 추상화 |
|------|--------|--------|
| **핵심 키워드** | 공통점 추출 | 핵심만 남김 (단순화) |
| **방향성** | 구체 → 일반 (상향식) | 상세 → 핵심 (하향식) |
| **반대 개념** | 특수화 (Specialization) | 구체화 (Concretization) |
| **결과물** | 상속, 부모 클래스 | 인터페이스, 추상 메서드 |

---

## 3. 예시

### 일반화 예시

원, 사각형, 삼각형이라는 구체적인 클래스가 있을 때:

```
Circle    → 너비, 둘레 계산 가능
Rectangle → 너비, 둘레 계산 가능
Triangle  → 너비, 둘레 계산 가능
```

공통점을 추출하여 `Shape`라는 상위 클래스를 만듭니다:

```java
// 일반화의 결과: 부모 클래스
class Shape {
    double getArea();
    double getPerimeter();
}

class Circle extends Shape { ... }
class Rectangle extends Shape { ... }
class Triangle extends Shape { ... }
```

### 추상화 예시

**자동차 운전:** 운전자가 엔진의 작동 원리를 몰라도 브레이크와 가속 페달만으로 운전할 수 있습니다. 복잡한 내부 구현을 숨기고 핵심 인터페이스만 노출한 것입니다.

**온라인 강의 시스템:** 학생을 모델링할 때 키, 혈액형 등은 제외하고 이름과 학번만 남깁니다. 해당 도메인에서 필요한 핵심 속성만 추출한 것입니다.

```java
// 추상화의 결과: 인터페이스
interface Drivable {
    void accelerate();
    void brake();
    // 엔진 내부 동작은 숨김
}
```

## 4. 설계 관점에서의 관계

일반화와 추상화는 **대립하는 개념이 아니라 상호보완적**입니다.

1. **일반화**로 구체적인 대상들의 공통점을 정리하여 개념 구조를 형성합니다
2. **추상화**로 사용자에게 필요한 핵심 역할만 노출합니다
3. 결과적으로 시스템의 **복잡도를 낮추고 의존성을 줄입니다**

```
구체적 대상들
    ↓ (일반화: 공통점 추출)
상위 개념/부모 클래스
    ↓ (추상화: 핵심만 노출)
인터페이스/추상 메서드
```

---

## 5. 실무 적용

### 일반화가 필요한 상황

- 여러 클래스에 중복 코드가 발생할 때
- "is-a" 관계가 명확할 때 (Circle is a Shape)
- 공통 동작을 부모 클래스로 올려 재사용하고 싶을 때

### 추상화가 필요한 상황

- 구현 변경이 클라이언트에 영향을 주지 않아야 할 때
- 외부에 최소한의 인터페이스만 노출하고 싶을 때
- 의존성 역전 원칙(DIP)을 적용할 때
