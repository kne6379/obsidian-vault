# business 영역 규칙

## 영역 소개

| 영역 | 주요 주제 |
|------|-----------|
| 브레인스토밍 | 시장 조사, 아이디어 발산, 평가 |
| 사업 기획 | 비즈니스 모델, 시장 분석, 경쟁 분석 |
| 실행 계획 | MVP 설계, 로드맵, 리소스 계획 |

## 문서 유형별 구조

### 브레인스토밍 문서 (brainstorms/)

1. 주제 정의 - 범위와 방향
2. 시장 조사 - 시장 현황, 주요 플레이어, 트렌드, 페인 포인트
3. 아이디어 목록 - 도출된 사업 아이디어 상세
4. 아이디어 평가 - 실현 가능성, 시장성, 차별성 평가
5. 추천 및 다음 단계 - 상위 아이디어와 후속 액션
6. 관련 문서
7. 참고 자료 - 시장 조사에 활용한 출처

## 메타데이터

브레인스토밍 문서는 `domain` 필드를 포함합니다:

```yaml
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [brainstorm, 주제태그]
status: idea | exploring | validated | archived
domain: 대상 산업/분야
```

## 상태 정의

| 상태 | 설명 |
|------|------|
| idea | 초기 아이디어 단계 |
| exploring | 추가 조사/검증 진행 중 |
| validated | 시장성/실현 가능성 검증 완료 |
| archived | 보류 또는 폐기 |

## 아이디어 평가 기준 (5점 척도)

| 기준 | 1점 | 3점 | 5점 |
|------|-----|-----|-----|
| 실현 가능성 | 기술/리소스 확보 어려움 | 일부 도전적 | 기존 기술로 구현 가능 |
| 시장성 | 니치 시장, 불확실한 수요 | 성장 중인 시장 | 대규모 시장, 명확한 수요 |
| 차별성 | 기존 솔루션과 유사 | 부분적 차별화 | 명확한 경쟁 우위 |

## 태그 체계

- 유형: brainstorm, plan, analysis
- 주제: ai, llm, saas, automation, market, competitor, trend, business-model, mvp, revenue
- 상태: idea, exploring, validated, archived

## 작성 스타일 세부

- 아이디어 설명은 간결하고 핵심적으로 작성합니다.
- 시장 규모, 성장률 등 수치는 출처를 명시합니다.
- 추정치는 근거를 함께 제시합니다.
- "~인 것 같습니다" 대신 "~로 분석됩니다", "~로 추정됩니다"를 사용합니다.
- 아이디어는 추상적이 아닌 실행 가능한 수준으로 작성합니다.
- 타겟 고객, 핵심 가치, 수익 모델을 반드시 포함합니다.

## 영역 간 링크 규칙

- develop/ 영역의 관련 기술 문서와 적극적으로 연결합니다.
- work/ 영역의 실무 경험과 연결 가능하면 링크합니다.
- 동일 영역 내 유사 브레인스토밍 문서끼리도 연결합니다.

## 작업 전 필수 확인

- 새 문서 작성 시 `business/_templates/BRAINSTORM.md` 템플릿을 기반으로 합니다.
- 새 문서 작성 전 반드시 `business/_meta/INDEX.md`를 읽어 중복을 방지합니다.
- 새 용어 작성 시 `business/_meta/GLOSSARY.md`를 확인합니다.
